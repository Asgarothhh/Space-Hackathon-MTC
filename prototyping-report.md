## 2. Выводы по прототипированию

### 2.1 Готовность API

- **Контракт стабилен:** все фазы (examples, coverage, stateful) проходят без падений.
- **Спецификация согласована с реализацией:** бэкенд соответствует OpenAPI-описанию для протестированных сценариев.
- **Прототип пригоден для дальнейшей разработки:** базовая логика эндпоинтов проверена автоматически.

### 2.2 Ограничения прототипа

| Проблема | Затронуто операций | Рекомендация |
|----------|--------------------|--------------|
| **Аутентификация** | 13 | Настроить `--auth` или `-H "Authorization: Bearer <token>"` для проверки защищённых эндпоинтов |
| **Отсутствие тестовых данных** | 12 | Добавить в `schemathesis.toml` реальные `project_id`, `user_id`, `server_id` для admin-операций |
| **Расхождение схемы и валидации** | 30 | Ужесточить ограничения в OpenAPI (форматы, min/max) или ослабить валидацию в Pydantic |

### 2.3 Исключённые операции

- **3 из 38** операций не тестируются (тег `agent`): `/ask`, `/file_loader`, `/health` — на этапе прототипа считаются экспериментальными.

### 2.4 Stateful-тестирование

- Покрыто **5 из 46** API Links (связей между операциями).
- 117 сценариев цепочек вызовов пройдено.
- Для более глубокой проверки workflow нужны тестовые данные и аутентификация.

---

## 3. Рекомендации для продакшена

1. **Аутентификация:** настроить тестового пользователя и передавать JWT в Schemathesis.
2. **Тестовые данные:** создать фикстуры (пользователи, проекты, VM) и указать их в конфиге.
3. **Схема:** привести OpenAPI в соответствие с реальной валидацией (форматы, границы чисел, обязательные поля).
4. **Agent-эндпоинты:** после стабилизации — включить в контрактные тесты, убрав `--exclude-tag agent`.
